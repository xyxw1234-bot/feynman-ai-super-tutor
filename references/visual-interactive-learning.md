# 可视化互动学习增强标准

## 目标

让费曼学习过程在数理化等图示密集场景中更直观，但不偏离“学生重新讲清楚”的核心。图示、配图、H5 和小游戏都是学习脚手架，不是答案替代品。

## 触发判断

优先考虑视觉或互动素材的信号：

- 学生能背定义但解释不出图像变化。
- 学生卡在方向、受力、运动趋势、回路路径、浮力变量、实验步骤。
- 学生连续两轮文字解释仍不稳定。
- 知识点天然依赖图表、空间结构、微观粒子或过程变化。

不应生成素材的信号：

- 一个短追问即可暴露并修正问题。
- 用户明确只要文字提示。
- 当前素材需要外部版权图片且用户未提供合法来源。
- 智能体无法验证学科正确性或页面质量。

## 输出分级

1. 文本追问：学生基本能讲，只需一个迁移问题。
2. 简图/步骤图：静态结构、步骤关系、一个易错点。
3. 互动 H5：变量可拖动、图像会变化、方向/回路/实验过程需要操作。
4. 小游戏：低龄巩固、复习辨析，但必须围绕一个知识点。

## H5 产品标准

- 浅色背景，移动端优先。
- 首屏一句话说明学习目标。
- 至少一个真实控件：滑块、按钮、拖动或选择。
- 操作后有即时反馈。
- 页面只服务一个知识点。
- 必须有“回到聊天复述”的任务。
- 不出现内部路径、安装命令、测试说明、无关品牌或版权不明素材。

## 发出前质量检查

1. `feynman_check_visual_asset` 静态检查通过。
2. 浏览器能打开页面，无脚本错误。
3. 360px 手机宽度无横向溢出，控件可点击。
4. 所有滑块/按钮操作后画面和反馈真实变化。
5. 学科正确性复核：数学图像、物理方向、化学粒子、实验边界不能误导。
6. 部署后公网链接 HTTP 200。
7. 发给学生时，明确让学生操作后回到聊天复述。

## 推荐话术

- “你现在卡的不是定义，而是变量怎么变。我给你做一个可以拖动的小实验，你操作后回来讲给我听。”
- “这个点用图会更清楚。我先做一个简图，之后你再用自己的话解释。”
- “页面只是帮助你观察，最后还是要你自己讲清楚。”
